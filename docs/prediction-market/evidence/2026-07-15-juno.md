## Sign-off refresh procedure

At architecture sign-off and immediately before deployment planning:

1. choose one finalized height and record its chain ID, block hash, time, and app hash;
2. send every state query with x-cosmos-block-height set to that height and preserve the returned height header;
3. repeat governance, consensus, wasm policy, module account, contract info, code info, and oracle config queries;
4. compare every value to the accepted ADRs; any checksum/admin/address/timing mismatch blocks planning;
5. archive untouched response bodies and response headers in the review system;
6. independently query a second RPC/LCD provider and record disagreement rather than averaging it away.
